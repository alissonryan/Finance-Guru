## Slide Outline

1. **Cover**
   - Title, subtitle, date, compliance disclaimer (educational only)
   - Branding per Finance Guru style (clean, minimalist)
2. **Executive Summary**
   - 3-4 bullets: objectives, key findings, recommended action
   - Highlight risk posture and timeframe
3. **Market & Macro Context**
   - Rates, volatility regime, liquidity signals
   - Charts with sources + timestamps
4. **Research Highlights**
   - Security/sector insights, peer comparisons
   - Data table or visual + citation
5. **Quantitative Analysis**
   - Metrics: Sharpe, Sortino, drawdown, VaR 95/99
   - Scenario outcomes (bull/base/bear) with graph/table
6. **Strategy Recommendations**
   - Margin/dividend/cash-flow tactics mapped to user constraints
   - Benefits, risks, monitoring triggers
7. **Implementation Plan**
   - Timeline, milestones, responsible agents/tasks
   - Required artifacts (models, research follow-ups)
8. **Risk & Compliance**
   - Margin buffers, liquidity checks, concentration notes
   - Compliance reminders, limitations
9. **Next Steps**
   - Action checklist with owners & due dates
   - Optional callouts for additional research
10. **Appendix**
    - Sources & assumptions table (with timestamps)
    - Detailed charts, sensitivity tables, glossary

_Add/delete slides as needed but maintain flow from insight → strategy → execution → compliance._

## Visual Standards

- Consistent color palette (muted blues/greens, highlight risk in amber/red)
- Use no more than 3 fonts; prefer system defaults for portability
- Each chart/table must cite source + "As of" date
- Include scenario labels on charts and explain drivers in captions

## Compliance & Disclaimers

- Repeat educational disclaimer in footer of every slide
- Highlight when data is modeled vs. actual
- Note any assumptions requiring user confirmation
- Encourage consulting a licensed advisor before acting

## Delivery Notes

- Provide speaking notes for each slide (1-2 bullets) in presenter view or doc appendix
- Export final deck to PPTX and PDF if requested
- Store file under `/finance/reports/` unless user specifies `[save:/path]`
